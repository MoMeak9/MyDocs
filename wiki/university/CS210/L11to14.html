Advantages:

  1. Simple to implement.
  2. Hash table never fills up, we can always add more elements to the chain.
  3. Less sensitive to the hash function or load factors.
  4. It is mostly used when it is unknown how many and how frequently keys may be inserted or deleted. Disadvantages:
  5. Cache performance of chaining is not good as keys are stored using a linked list. Open addressing provides better cache performance as everything is stored in the same table.
  6. Wastage of Space (Some Parts of hash table are never used)
  7. If the chain becomes long, then search time can become O(n) in the worst case.
  8. Uses extra space for links.
